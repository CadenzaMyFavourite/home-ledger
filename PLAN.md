## 2. 已确定的关键决策

1. 使用 Tauri 2 + React + TypeScript + Vite。
2. React 不直接执行 SQL；所有数据读写经类型化 Tauri commands 进入 Rust 应用层。
3. Rust 使用 SQLx + SQLite，迁移由 SQL 文件管理并在应用启动时执行。
4. 金额全部使用 `INTEGER` 最小货币单位；多币种换算也保存整数结果和有理数汇率，不使用浮点数。
5. 交易、事件、附件、周期规则分别存储，使用关联表连接；不使用一个万能 JSON 表替代核心关系模型。
6. 实际统计只包含未删除、`completed`、`income|expense` 的交易；`planned`、`pending`、`cancelled` 和 `transfer` 不进入实际收支。
7. 本地 AI 是可拔除适配器，只接收明确授权的数据快照；AI 不能执行 SQL、修改金额或绕过人工确认。
8. 删除默认采用软删除；分类、支付方式等被历史记录引用时只能停用，不能破坏引用。
9. 日期边界由 Rust 按用户时区计算；SQL 接收已经确定的半开区间，不自行猜测时区。
10. 备份格式采用带 manifest 与校验和的版本化归档；恢复前强制创建恢复点并要求确认。

更完整的理由与边界见 [ARCHITECTURE.md](./ARCHITECTURE.md)，表结构见 [DATA_MODEL.md](./DATA_MODEL.md)。

## 3. MVP 边界

### 3.1 包含

- 本地 SQLite、版本化迁移、示例数据模式。
- 收入、支出、转账 CRUD 与批量操作。
- 分类/子分类、支付方式、地点、家庭成员、标签、税务标签。
- 月历、普通/重要事件、每日详情、事件与交易关联。
- 周期规则、计划交易、提醒和桌面通知。
- 月度/年度确定性统计、分类统计与对比。
- CSV 导入/导出、Excel/PDF 报告、JSON 完整备份/恢复。
- 常用建议、保存筛选、交易模板。
- Ollama 与 OpenAI-compatible 本地端点（覆盖 LM Studio）。
- AI 月度总结、可审核建议、税务候选清单和年度资料包。
- 中英文、浅色/深色模式、键盘操作、完整状态反馈。
- 50,000 条交易的性能验收、自动化测试、Windows 安装包。

### 3.2 不包含

- 银行聚合或自动登录、保存完整卡号或银行密码。
- 云同步、多人实时协作、手机 App。
- 自动报税提交或确定某项支出可以抵税。
- 全自动 OCR、投资组合、支付处理。
- 未经用户明确启用的外部网络调用或遥测。

## 8. 性能预算（MVP 初始值）

- 50,000 条交易下，默认列表首屏可交互：目标 `< 1.5s`（发布构建、基准 PC）。
- 常用筛选/排序数据库查询：P95 `< 250ms`。
- 单月汇总：P95 `< 150ms`；单年汇总：P95 `< 300ms`。
- UI 表格一次只渲染可视行，数据库分页默认 100 行。
- CSV 导入和导出在后台任务执行，UI 保持响应并显示进度。
- 附件不加载进数据库或列表响应；缩略图按需生成和缓存。

这些是工程预算，不是未经测量的承诺；Phase 2 建立基准后记录测试机器并调整。

## 9. 主要风险与缓解

| 风险                     | 缓解                                                           |
| ------------------------ | -------------------------------------------------------------- |
| 多币种被错误相加         | 保存基础币种整数金额；缺少换算时明确标记并禁止进入单一币种汇总 |
| DST/月末导致周期漂移     | recurrence 带 IANA 时区；按本地日历生成，再转换为 UTC          |
| 重复启动生成多笔账单     | `recurring_occurrences` 唯一键 + 单事务物化                    |
| 前端被攻破后读取任意文件 | 最小 Tauri capability；附件操作只接受逻辑 ID，不接受任意路径   |
| 备份看似成功但不可恢复   | manifest、校验和、`integrity_check`、恢复演练                  |
| AI 幻觉污染财务          | 聚合由程序算；AI 结果单独存储；建议必须审核                    |
| 50k 表格卡顿             | 数据库分页、索引、虚拟行、避免全量 IPC                         |
| 配置删除破坏历史         | 停用代替硬删除，外键 `RESTRICT`                                |
| Windows 后台提醒预期不清 | MVP 明示启动/恢复补查语义；系统级后台调度另立里程碑            |

## 10. 最终完成定义

- 全部迁移可从空库顺序执行，且升级路径经过备份/恢复测试。

- README 覆盖开发、测试、构建、AI、导入导出、备份恢复和免责声明。
